21. Optimized Index (.cspt) -- Prefix Collisions and Fallback

These tests exercise the .idx.oft.cspt optimized index: the fast lookup path it accelerates, prefix-collision correctness, silent fallback to .idx.oft when the optimized index is absent, and regeneration when the underlying .idx.oft is rebuilt.

Tests C and D require host PC access to the SD card (device powered off, card removed) to delete specific files. This mirrors the convention already used in Ch 3 Test M.

Test A: Baseline lookup on a dictionary with optimized index

The english-full dictionary ships with english-full.idx.oft.cspt. Every lookup must return the correct definition. This implicitly exercises the optimized-index fast path because the device tries it first when present.

Dictionary

english-full

Words

  • apple
  • bridge
  • cloud
  • dawn
  • echo

Steps

  1. Select english-full in the picker. No preparation should be required.
  2. Look up each word above.
  3. Each returns a valid definition. No errors. No crash.

Test B: Prefix-collision disambiguation

The prep-cspt-prefix-collision dictionary contains headwords that share 16-character prefixes. Each word must still return its own distinct definition -- not a sibling's. Requires Ch 4 Test F to have been run so this dictionary has been prepared.

Dictionary

prep-cspt-prefix-collision

Words

16-char prefix international:

  • internationalism
  • internationalize
  • internationalized

16-char prefix microelectromech:

  • microelectromechanical
  • microelectromechanics

Control words (no collision):

  • antidisestablish
  • simple
  • test

Steps

  1. Select prep-cspt-prefix-collision in the picker. If prep steps are listed, Ch 4 Test F was skipped -- run that first, then return here.
  2. Look up each word above.
  3. Each word returns its own distinct definition. Compare against the definition text in test/data/dictionary-sources/prep-cspt-prefix-collision.json if in doubt.
  4. No two colliding words return the same definition.

Test C: Fallback to .idx.oft when optimized index is absent

Lookups must still work when .idx.oft.cspt is missing. Re-selecting the dictionary afterwards must re-offer the Generate optimized index prep step.

Dictionary

english-full

Setup (on device, before the pre-condition)

  1. Open Settings -> Dictionary. Confirm english-full is the currently selected dictionary (Test A already does this). If it is not, select it now and let any preparation finish. This must happen before deleting .idx.oft.cspt on the host -- otherwise selecting english-full later would trigger prep and automatically regenerate the .cspt, making the fallback path unobservable.

Pre-condition (on host PC)

  1. Power the device off. Remove the SD card.
  2. On the host, delete /.dictionaries/english-full/english-full.idx.oft.cspt (or the equivalent under /dictionaries/). Leave english-full.idx.oft in place.
  3. Re-insert the SD card. Power on.

Steps

  1. Look up each Test A word from within this test epub: apple, bridge, cloud, dawn, echo. Each returns the correct definition. No errors. (This exercises the .idx.oft fallback path since .cspt is absent.)
  2. Open Settings -> Dictionary. Re-select english-full.
  3. Confirmation screen lists exactly one step: Generate optimized index.
  4. Press Confirm. Step completes: [OK].
  5. Preparation complete.
  6. Return to this test epub and look up the Test A words again: apple, bridge, cloud, dawn, echo. Each returns the correct definition -- confirms the newly regenerated .idx.oft.cspt is functional on the fast path.

Test D: Stale optimized index regenerated when .idx.oft is regenerated

Deleting .idx.oft while leaving .idx.oft.cspt in place must cause the device to regenerate both on next prep -- the stale optimized index is wiped and rebuilt.

Dictionary

english-full

Pre-condition (on host PC)

  1. Power the device off. Remove the SD card.
  2. On the host, delete english-full.idx.oft but leave english-full.idx.oft.cspt in place.
  3. Re-insert the SD card. Power on.

Steps

  1. Open Settings -> Dictionary. Re-select english-full.
  2. Confirmation screen lists in order (this is the key signal that the stale .cspt was detected and scheduled for rebuild):
    • Generate dictionary offset file
    • Generate optimized index
  3. Press Confirm. Both steps run sequentially, each showing [ > ] then [OK].
  4. Preparation complete.
  5. Return to this test epub and look up the Test A words: apple, bridge, cloud, dawn, echo. Each returns the correct definition -- confirms both the regenerated .idx.oft and .idx.oft.cspt are functional.
