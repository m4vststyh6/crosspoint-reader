19. IPA Phonetic Characters

This chapter tests rendering of IPA phonetic characters in dictionary definitions. Set the active dictionary to ipa-phonetic before starting.

IPA characters must render as legible Doulos SIL glyphs. A filled diamond indicates a rendering failure.

Test A: IPA Extensions block (U+0250 to U+02AF)

Look up ipa-extensions.

Expected: definition shows a heading followed by a dense block of 96 IPA glyphs. All must be legible Doulos SIL characters, not filled diamonds.

Test B: Modifier Letters block (U+02B0 to U+02FF)

Look up ipa-modifier-letters.

Expected: definition shows a heading followed by a dense block of 80 glyphs. All must be legible.

Test C: Phonetic Extensions block (U+1D00 to U+1D7F)

Look up phonetic-extensions.

Expected: definition shows a heading followed by 128 glyphs. All must be legible.

Test D: Phonetic Extensions Supplement (U+1D80 to U+1DBF)

Look up phonetic-extensions-supplement.

Expected: definition shows a heading followed by 64 glyphs. All must be legible.

Test E: Mixed IPA and regular text (inline font switching)

Look up pronunciation.

Expected: the definition shows a pronunciation transcription in IPA symbols inline within regular English text. The IPA characters must be legible and match the surrounding text size visually. No filled diamonds.

Test F: Mixed IPA mid-sentence

Look up schwa.

Expected: definition mixes regular English text with inline IPA symbols. IPA glyphs must appear correctly inline. The example transcriptions in brackets must be fully legible.

Test G: Word-wrap with long IPA string

Look up ipa-wrap-test.

Expected: the long IPA string in the definition wraps correctly at the screen margin. No truncation or corruption of characters at the wrap point. All glyphs must remain legible after wrapping.

Test H: Pronunciation line with no trailing glyph (F-060/F-061)

These entries use the real-dict format: plain-text pronunciation on the first line followed by a newline, then HTML body. The newline must be treated as a line break, not rendered as a filled diamond.

Look up meetings (stems to meeting).

Expected: pronunciation line /ˈmitɪŋ/, /ˈmiːtɪŋ/ appears cleanly with no trailing filled diamond. Body text (Noun definition and etymology) follows on the next line.

Look up austere.

Expected: pronunciation line /ɑˈstiɹ/, /ɒˈstɪə(ɹ)/, /ɔˈstiɹ/, /ɔːˈstɪə(ɹ)/ appears cleanly with no trailing filled diamond. Unsupported Greek characters in the etymology body render as filled diamonds — this is expected and correct behaviour for unsupported scripts.

Test I: Word-select on an IPA definition

Word-select mode must work correctly on a definition that contains IPA characters. Highlight boxes must align accurately over the words they represent, including over runs of IPA glyphs.

Steps

  1. Look up pronunciation. Definition screen appears with mixed IPA and regular text.
  2. Press Look Up Word to enter word-select mode.
  3. Verify the initial highlight is approximately centred in the definition text.
  4. Navigate through the definition. Verify each highlight box sits directly over its word — no offset for either regular text or IPA character runs.
  5. Navigate to any regular English word in the definition and press Confirm. A lookup is performed. No crash.
  6. Long-press Back to return to the reader.

Test J: Individual Latin and Greek characters used in IPA

Look up ipa-latin-greek.

Expected: the definition shows each of the nine individual Latin and Greek characters classified as IPA by the renderer: æ (U+00E6), ð (U+00F0), ø (U+00F8), ħ (U+0127), ŋ (U+014B), œ (U+0153), β (U+03B2), θ (U+03B8), χ (U+03C7). All must render as Doulos SIL glyphs — no filled diamonds. The mixed-text paragraph must show smooth inline font switching between regular and IPA fonts.

Test K: Combining marks on IPA and non-IPA bases

Look up ipa-combining-marks.

Expected: the definition shows IPA base characters with combining diacritics from all three supported ranges — Combining Diacritical Marks (U+0300–036F), Combining Diacritical Marks Supplement (U+1DC0–1DFF), and Combining Diacritical Marks for Symbols (U+20D0–20FF). Combining marks on IPA bases must inherit IPA status and render with the Doulos SIL font. Combining marks on regular Latin bases must render with the definition font (non-IPA). No combining mark should appear as a standalone glyph or filled diamond. Supplement and symbol range marks may lack a Doulos SIL glyph; a filled diamond for those is a font coverage issue, not a classification bug.
